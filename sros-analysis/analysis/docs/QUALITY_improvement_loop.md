# 결과 품질 반복 개선 (Quality Improvement Loop)

**목표:** 최종 결과(08_results_report)의 품질을 **100점 만점 100점 수준**으로 달성하고, 분석 예시를 통해 **이상적 분석 수행 결과 리포트 시나리오**가 나오도록 전체 프로젝트를 반복 개선한다.

---

## 1. 품질 수준 수치화

| 수준 | 점수 구간 | 설명 |
|------|-----------|------|
| 기초 | 0–49 | 서론·본론·결론 구분 없음 또는 근거 미표기, 수치 출처 불명. |
| 보통 | 50–74 | 구분은 있으나 근거가 부분적, 결론이 본론 수치를 직접 인용하지 않음. |
| 양호 | 75–89 | 서론·본론·결론 명료, 대부분 근거 표기. 결론 인용 또는 다음 단계 구체성 일부 부족. |
| **이상적 (목표)** | **90–100** | 서론·본론·결론 명료, **모든 주장에 근거(문서+절)** 표기, 결론이 본론 수치·문서 직접 인용, 다음 단계 A/B/C 구체적, 재현성·참고 문서 완비. |

- **루브릭:** design/08_report_quality_rubric.md (영역별 배점·100점 달성 조건).
- **현재 수준 진단:** 08 작성 후 루브릭으로 자가 채점하여 점수 구간을 확인한다.

---

## 2. 반복 개선 절차

1. **08 작성** — design/templates/08_results_report.md 구조(서론·본론·결론·근거) 준수.
2. **자가 채점** — 08_report_quality_rubric.md §4 체크리스트 및 영역별 배점으로 채점.
3. **미달 항목 보완** — 근거 누락, 결론 인용 부족, 다음 단계 모호함 등을 수정.
4. **재채점** — 90점 이상(이상적) 달성 시 1사이클 종료. 미달 시 2–3 반복.

---

## 3. 이상적 리포트 시나리오 (예시)

- **예시:** analysis/examples/expansion/08_results_report.md.
- **구성:** 서론(요약·연구 목적·질문·데이터 범위·주요 결과·해석 원칙, 각 항목에 **근거:** 01_charter, 02 §…, 06 …) → 본론(데이터·가설·결과·한계, 각 절에 **근거:** 02/03/04/05/06/07) → 결론(본론 수치·문서 직접 인용 요약, A/B/C 구체적).
- **목표:** 새 분석 실행 시에도 위와 같은 **구성 명료·근거 명확** 리포트가 나오도록, 템플릿·규칙·스킬·서브에이전트가 루브릭과 연동되어 있다.

---

## 4. 프로젝트 전반 반복 개선 시 참조

- **규칙:** R9 (08 품질·근거·루브릭 100점 목표).
- **스킬:** 보고(전 단계) — 08_report_quality_rubric, 자가 채점, 100점 목표.
- **서브에이전트:** Level 5 태스크 — 루브릭 자가 채점·미달 보완.
- **템플릿:** design/templates/08_results_report.md (서론·본론·결론·근거 필드).
- **예시:** examples/expansion/08_results_report.md (이상적 시나리오).

---
*품질 개선은 08 루브릭 자가 채점 → 미달 보완을 반복하여 100점 수준을 유지하는 루프로 수행한다.*
