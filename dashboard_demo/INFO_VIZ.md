# 정보 시각화(Information Visualization) 검증

대시보드의 폰트, 색상, 컴포넌트 크기가 정보 시각화 이론 관점에서 적절한지 Self-check에서 검증한다.

## 검증 항목

### 1. 폰트·폰트 크기
- **본문 폰트 크기**: 11pt~14pt 수준 권장 (가독성). Self-check에서 `body`의 `fontSize` 계산값이 10px~16px 범위인지 확인.
- **피벗 테이블**: 11pt, 라인당 세로 간격 콤팩트(줄간격 1.35 등)로 설정되어 있는지 확인.

### 2. 색상
- **텍스트/배경 대비**: 본문 텍스트 색상과 배경 색상이 존재하여 가독성이 확보되는지 확인.
- **메트릭 강조**: 선택된 메트릭 버튼·KPI 값 등에 `--metricColor` 사용으로 일관된 강조.

### 3. 컴포넌트 크기
- **탑바·버튼**: 오버플로우 없이 한 줄에 배치되는지.
- **필터 접기 버튼**: 높이 32px 이하, 아이콘+텍스트 인라인.
- **피벗 닫기 버튼**: X 표시, 오른쪽 위, 적절히 큰 크기(예: 36px).

### 4. Self-check 항목 요약
- L1 가로 오버플로우
- topbar vs main 겹침 없음
- 우측 패널 접기 시 가로 오버플로우 없음
- 차트 존재 여부
- **정보시각화: 본문 폰트 크기(11~14pt 권장)**
- **정보시각화: 텍스트/배경 대비 존재**
- KPI vs Pivot 합계 일치
- 기타 레이아웃·접근성 항목

## 적용 원칙
- **제한된 채널**: 한 화면에 핵심 메트릭 3종(판매량, 스마트 판매량, 연결 수)만 선택 가능.
- **계층 구조**: 지역(글로벌 > 지역 > 법인 > 국가), 제품(제품군 > 레벨2 > 레벨3)을 독립적으로 필터.
- **일관된 데이터 소스**: Pivot과 차트가 동일한 `getFilteredData()` 기반으로 집계.
